# Практична робота №5: Клієнтський веб-додаток (JavaScript + API)

**Варіант 2:** Вітрова електростанція.

## Про проект
Це клієнт-серверний веб-додаток для моніторингу вітрової електростанції (10 турбін) у режимі реального часу. Проєкт виконано згідно з вимогами до практичної роботи №5, з фокусом на взаємодію з WebSocket API та побудову інтерфейсу для динамічної візуалізації даних.

**Основний функціонал:**
- Отримання даних від сервера (швидкість та напрямок вітру, загальна потужність, потужність турбін та КВВП) через WebSocket кожні 2 секунди.
- Динамічне оновлення поточних метрик на інформаційній панелі (статус підключення, загальна потужність, швидкість та напрямок вітру).
- Візуалізація даних у реальному часі за допомогою 4-х типів діаграм (Chart.js):
  - Стовпчаста діаграма (потужність кожної з 10 турбін).
  - Лінійний графік (швидкість вітру).
  - Радарна діаграма / Роза вітрів (напрямок вітру).
  - Gauge-індикатор у вигляді півкола (коефіцієнт використання встановленої потужності - КВВП).
- Ведення та автоматичне оновлення таблиці з історією останніх вимірювань.

## Використані технології
- **HTML5:** семантична структура дашборду.
- **CSS3 / Bootstrap 5.3:** адаптивний дизайн, сітка для карток та розміщення графіків.
- **JavaScript (ES6+):** клієнтська логіка, робота з об'єктом WebSocket.
- **Chart.js:** бібліотека для відмальовування динамічних графіків на `canvas`.
- **Node.js / ws:** середовище виконання та бібліотека для створення тестового WebSocket-сервера.

## Інструкція з запуску
1. Встановити необхідні пакети: `npm install ws express cors`
2. Запустити локальний сервер: `node server.js`
3. Відкрити файл `index.html` у браузері.
