# Практична робота №4: Моніторинг енергетичних об'єктів у реальному часі

**Варіант 2:** Вітрова електростанція.

## Про проект
Це клієнт-серверний веб-додаток для моніторингу вітрової електростанції (10 турбін) у режимі реального часу. Проєкт виконано згідно з вимогами до практичної роботи №4, з фокусом на організацію двостороннього зв'язку за допомогою протоколу WebSocket та динамічну візуалізацію даних.

**Основний функціонал:**
- Симуляція роботи вітрової станції на сервері (генерація швидкості вітру, напрямку, потужності турбін та КВВП).
- Безперервна передача даних від сервера до клієнта кожні 2 секунди через WebSocket без необхідності HTTP-запитів.
- Динамічне оновлення метрик на сторінці (статус підключення, загальна потужність).
- Візуалізація даних у реальному часі за допомогою 4-х типів діаграм:
  - Лінійний графік (швидкість вітру).
  - Стовпчаста діаграма (потужність кожної з 10 турбін).
  - Gauge-індикатор у вигляді півкола (коефіцієнт використання встановленої потужності - КВВП).
  - Радарна діаграма / Роза вітрів (напрямок вітру).
- Ведення та автоматичне оновлення таблиці з історією останніх вимірювань.
- Обробка втрати з'єднання з автоматичною спробою перепідключення.

## Використані технології
- **HTML5:** семантична структура дашборду.
- **CSS3 / Bootstrap 5:** адаптивний дизайн, сітка для карток та розміщення графіків.
- **JavaScript (ES6+):** модульна клієнтська архітектура (`app.js`, `api.js`, `charts.js`), робота з WebSocket API.
- **Chart.js:** бібліотека для відмальовування динамічних графіків на `canvas`.
- **Node.js:** середовище виконання серверного коду.
- **ws:** бібліотека для створення WebSocket-сервера.
